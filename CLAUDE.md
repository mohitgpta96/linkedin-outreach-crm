## Hard Rules (NEVER violate)
1. Connection requests MUST be under 300 characters
2. NEVER use "I" more than twice in any message
3. NEVER sound salesy — tone is helpful, peer-to-peer
4. ALWAYS reference something specific about the prospect [SPECIFIC: marker required]
5. Follow-ups must add NEW value, never just "checking in"
6. All messages must pass scripts/validate-message.sh

## Workflow
1. Research prospect → save to prospects/researched/
2. Draft using templates/ as starting point
3. Personalize based on research
4. Validate against Hard Rules
5. Output to output/daily-batch/
